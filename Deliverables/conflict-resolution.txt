During this assignment, I used a Git feature-branch workflow to build a simple calculator application while practising proper version control and conflict resolution. I started by creating separate branches for two features: multiplication and division. The idea was to keep each feature isolated so that changes would not interfere with the main codebase.

On the multiplication branch, I implemented a multiply function with basic logging and updated the version to 1.1.0. On the division branch, I added a divide function that includes a safety check to prevent division by zero, and updated the version to 1.0.1. While working through the task, I initially had some confusion around branch creation, file locations, and how changes were reflected in GitHub, which helped me better understand how commits and branches actually track changes rather than just saving files locally.

A merge conflict was intentionally created when both branches modified the same sections of the file, including the version number and the main execution block. I resolved the conflict manually by reviewing both versions, keeping both function implementations, and updating the version to 1.2.0 to reflect the combined state of the project. I also ensured the main section correctly executed and tested all functions.
