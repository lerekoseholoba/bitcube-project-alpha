# Sprint 1 Summary

## Executive Summary

Sprint 1 focused on building the foundational conference room booking workflow for the system. The team successfully implemented core booking, cancellation, and room filtering functionality while partially completing equipment filtering and maintenance scheduling features. Despite several integration and synchronization challenges, the sprint remained on track and delivered significant business value.

## Key Accomplishments

- Successfully implemented conference room booking functionality.
- Added booking conflict detection and validation logic.
- Completed room capacity filtering functionality.
- Implemented booking cancellation workflow.
- Integrated frontend booking interface with backend APIs.
- Improved collaboration between frontend and backend developers during the sprint.
- Completed major QA testing for booking and cancellation workflows.

## Challenges Faced

- Equipment filtering became unstable when handling multiple selections.
- Maintenance scheduling logic conflicted with room availability calculations.
- Calendar synchronization delays created inconsistent refresh behaviour.
- API response structures changed during frontend integration.
- Mobile responsiveness issues required additional refinement.

## Process Improvements Identified

- Finalize API structures earlier to reduce frontend blockers.
- Begin integration testing earlier in the sprint lifecycle.
- Increase automated testing for validation-heavy features.
- Reduce scope expansion during implementation.
- Improve communication around backend changes impacting frontend development.
